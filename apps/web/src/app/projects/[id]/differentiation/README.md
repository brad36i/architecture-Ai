# Differentiation (차별성 검토)

## 개요

선행연구·기존기술과의 **차별점**을 점검하고 강화하는 페이지입니다. R&D 평가에서 "차별성"은 기술적 창의성·기대효과 점수에 직결되므로, 제출 전 자가 점검이 필수입니다.

## 왜 이렇게 만들었는지

### 3탭 구조 (점검표 → 강약점 → 보완 제안)

- **탭 1 차별성 점검표**: 항목별로 체크하고 자가점수를 부여. "내가 여기서 몇 점 받을 것 같지?"를 스스로 점검
- **탭 2 강점·약점 분석**: AI가 draft·선행연구 기반으로 SWOT 스타일 분석. 약점을 인지해야 보완 가능
- **탭 3 보완 제안**: 약점에 대한 **구체적 문구 제안**. "목표치 근거 부족" → "선행연구 A 인용하여 15%p 달성 가능성 서술"처럼 액션 가능한 형태

### 점검 항목 구성

- **기술적 독창성**: 선행기술 대비 차별적 요소
- **시장 차별화**: 시장 Needs와 연계
- **정량적 근거**: 목표치·지표의 구체성
- **객관적 입증 방법**: 제3자 검증(시험성적서 등) 제시

→ 공고 평가 항목과 매핑되도록 설계

### 강점/약점 카드 색상

- 강점: 녹색(emerald) 계열
- 약점: 황색(amber) 계열
→ 한눈에 구분, 약점에 집중할 수 있음
