# 주제 선정 절차

## 1. 진입 시점

- 자유공모: 빈 프롬프트로 시작
- 지정공모: 연구주제 분석에서 선택한 주제와 RFP 내용이 0 노드에 담긴다
- 기회탐색형·구체화 옵션 기본 체크됨

## 2. Depth 구조

| Depth | 역할 |
|-------|------|
| 0 | 프롬프트 노드 (시작) |
| 1 | **실제 주제** 후보 (최대 3~5개) |
| 2~4 | 선택한 주제의 구체화·확장 |

## 3. Send → Depth 1 생성

- **기회탐색형**: RAG 벡터 DB에서 자료 검색, 빈틈 탐색 후 추천
- **구체화**: 연구과제 분석 페이지에서 선택한 해당 주제와 RFP 내용을 기반으로 구체화된 내용 생성

## 4. Depth 1 전용 기능

- **연구 합치기**: Depth 1 노드만 합칠 수 있음 (결과도 Depth 1)
- **+ 버튼**: 마음에 드는 주제가 없을 때, 기존과 다른 새 주제 추가 (반복 가능)
- **새로고침**: Pin 고정된 노드는 제외, 고정 내용을 참고해 새 노드 생성
- **위치**: Depth 1 전체를 아우르며, 수직 배치에서 맨 아래 Depth 1 노드 바로 아래
- **핵심**: 각 노드는 채팅 섹션이다. 독립적으로 작동된다. 

## 5. Depth 2~4

- +·새로고침 버튼 없음
- 선택한 Depth 1 주제를 단계적으로 구체화
- 마지막 노드: Depth 1~4 전체를 정리한 요약. 이때 최대 Depth가 4라는거지 그 이전에 마감할 수 있음. Depth 2부터 마감할 수 있음
- 연구계획서 작성 페이지로 이동하는 버튼 존재

## 6. 노드 공통 기능

- 내용 수정·보강(... 옵션에 배치)
- 선행연구 탐색 기능(... 옵션에 배치)

- 연구개발내용: 800자

# 연구주제 구체화 단계 로직

단순한 글자수 확장이 아니라, Depth마다 **다른 차원의 정보**를 추가한다.

| Depth | 추가되는 축 | 초점 |
|-------|-------------|------|
| **1** | What | 주제 (문제·대상·기여가 암묵적으로 담긴 한 문장) |
| **2** | Why | 근거·배경 (선행연구, 정책·수요, 왜 이 주제인가) |
| **3** | How | 방법론 (어떤 접근, 핵심 단계·모듈) |
| **4** | Evidence | 검증·산출물 (어떻게 증명할 것인가, 지표·실험) |
