# Execution Plan (실행계획 생성)

## 개요

연구계획서가 **선정된 후** 실제 연구수행을 위한 **일정계획**을 수립하는 페이지입니다. 제출 준비와는 별개로, "선정되면 어떻게 진행할까"를 Gantt 일정과 마일스톤으로 정리합니다.

## 왜 이렇게 만들었는지

### 실행계획의 역할

- 제출 체크리스트는 별도 페이지(submission-checklist)에서 처리
- 실행계획은 **선정 전략**이 아닌 **선정 후 실행**을 위한 일정·마일스톤 관리

### 2탭 구조 (Gantt → 마일스톤)

- **탭 1 Gantt 일정**: 연차×과제 일정을 테이블/바 형태로 표시. 도식/표 제안에서 생성한 Gantt와 연동 가능
- **탭 2 마일스톤**: 주요 이정표(중간보고, 최종보고 등)와 산출물(보고서, 특허 등) 목록. 시각적 카드로 "언제 뭘 낼지" 한눈에 파악

### Gantt 테이블 UX

- 연차별 12개월 컬럼 + 과제별 바(block)로 시각화
- 연구자가 일정 겹침·공백을 한눈에 파악
